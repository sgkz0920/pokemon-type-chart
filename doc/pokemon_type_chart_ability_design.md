## 2. 背景・目的

「ふゆう」や「あついしぼう」など、防御側ポケモンの**特性**はタイプ相性による最終倍率を変化させる。本追加設計では、防御側タイプに加えて特性を1つ選択できるようにし、より実戦に即した倍率を表示可能にする。

## 3. スコープ

### 3.1 対象とする特性

**「攻撃技のタイプのみに依存して倍率を変化させる防御側特性」**を対象とする（第9世代時点の効果に準拠）。

| No | 特性名 | 効果（本ツールでの扱い） | 補正の種類 |
|---|---|---|---|
| 1 | ふゆう | じめん技を無効化（0倍） | 無効化 |
| 2 | あついしぼう | ほのお・こおり技の倍率を0.5倍に補正 | 定率補正 |
| 3 | もらいび | ほのお技を無効化 | 無効化 |
| 4 | こんがりボディ | ほのお技を無効化 | 無効化 |
| 5 | ちょすい | みず技を無効化 | 無効化 |
| 6 | よびみず | みず技を無効化 | 無効化 |
| 7 | ちくでん | でんき技を無効化 | 無効化 |
| 8 | ひらいしん | でんき技を無効化 | 無効化 |
| 9 | でんきエンジン | でんき技を無効化 | 無効化 |
| 10 | そうしょく | くさ技を無効化 | 無効化 |
| 11 | かんそうはだ | みず技を無効化、ほのお技の倍率を1.25倍に補正 | 無効化＋定率補正 |
| 12 | もふもふ | ほのお技の倍率を2倍に補正（※） | 定率補正 |
| 13 | フィルター | 効果ばつぐん（倍率>1）の技の倍率を0.75倍に補正 | 条件付き補正 |
| 14 | ハードロック | 同上 | 条件付き補正 |
| 15 | プリズムアーマー | 同上 | 条件付き補正 |
| 16 | ふしぎなまもり | 効果ばつぐん（倍率>1）以外の技を無効化 | 特殊 |

※ もふもふの「直接攻撃の被ダメージ半減」は技のタイプ以外（接触判定）に依存するため対象外とし、ほのお2倍のみ反映する。この制限はUI上の注記では表示せず、本設計書の制限事項として記録する。

### 3.2 対象外とする特性（明示的除外）

| 分類 | 例 | 除外理由 |
|---|---|---|
| 技のタイプ以外の属性に依存 | ぼうだん（弾の技）、ぼうおん（音技）、ふゆうと異なる浮遊状態（テレキネシス等） | 技タイプだけでは判定不能 |
| ダメージ計算の他要素に作用 | マルチスケイル、こおりのりんぷん、パンクロック | 相性倍率ではなく最終ダメージへの補正 |
| 攻撃側の特性 | かたやぶり、テラボルテージ、フリーズスキン等 | 本ツールは防御側視点のため |
| 状況依存 | てんねん、ばけのかわ、テラスシェル（HP満タン時のみ）等 | 状態管理が必要で本ツールの範囲外 |

### 3.3 前提・制約

- 特性は**最大1つ**選択可能（実際のポケモンと同様）。デフォルトは「特性なし」。
- 特性の適用対象はあくまで防御側であり、逆引き機能（別拡張）とは独立している。

## 4. 相性計算仕様の変更

### 4.1 計算順序

最終倍率は以下の順に算出する（本体ゲームの仕様に準拠した近似）。

```
① タイプ相性倍率 = TYPE_CHART[攻撃][防御1] × TYPE_CHART[攻撃][防御2]
② 無効化判定     : 特性が攻撃タイプを無効化する場合 → 最終倍率 0（以降の補正は行わない）
③ 定率補正       : 特性が攻撃タイプに定率補正を持つ場合 → ①に係数を乗算
④ 条件付き補正   : フィルター系（③適用後の倍率が 1 より大きい場合のみ ×0.75）
⑤ ふしぎなまもり : ①の倍率が 1 以下の場合 → 0（倍率>1 の技のみ通す）
```

- ②の無効化はタイプ相性に優先する（例: ふゆう＋ひこうでもじめんは0倍のまま）。
- ④はタイプ相性で「効果ばつぐん（>1）」となった場合にのみ適用する。
- ⑤ふしぎなまもりは②〜④とは排他（同時に選択されることはない。1特性のみ選択可能なため）。

### 4.2 発生しうる倍率値の拡大

特性補正により、従来の6値（0 / 0.25 / 0.5 / 1 / 2 / 4）以外の倍率が発生する。

| 特性例 | 発生する倍率の例 |
|---|---|
| あついしぼう | 0.125（0.25×0.5）、1（2×0.5） など |
| フィルター系 | 1.5（2×0.75）、3（4×0.75） |
| かんそうはだ | 1.25、2.5、5（4×1.25） |
| もふもふ | 8（4×2） |

このため、結果表示の**固定6セクション方式を廃止し、動的グルーピング方式に変更**する（6章参照）。

## 5. データ構造設計

### 5.1 特性定義 `ABILITIES`

特性の効果は**宣言的ルール（rulesの配列）**で表現し、計算ロジックから分離する。

```js
// ルール種別:
//   immune       : attackTypes の技を無効化（最優先）
//   scale        : attackTypes の技の倍率に factor を乗算
//   scaleIfSuper : タイプ相性倍率が1より大きい場合のみ factor を乗算
//   wonderGuard  : タイプ相性倍率が1以下の技を無効化
const ABILITIES = [
  { id: "none",         name: "特性なし",       rules: [] },
  { id: "levitate",     name: "ふゆう",         rules: [{ kind: "immune", attackTypes: [8] }] },          // じめん
  { id: "thick-fat",    name: "あついしぼう",   rules: [{ kind: "scale",  attackTypes: [1, 5], factor: 0.5 }] }, // ほのお・こおり
  { id: "flash-fire",   name: "もらいび",       rules: [{ kind: "immune", attackTypes: [1] }] },
  { id: "well-baked",   name: "こんがりボディ", rules: [{ kind: "immune", attackTypes: [1] }] },
  { id: "water-absorb", name: "ちょすい",       rules: [{ kind: "immune", attackTypes: [2] }] },
  { id: "storm-drain",  name: "よびみず",       rules: [{ kind: "immune", attackTypes: [2] }] },
  { id: "volt-absorb",  name: "ちくでん",       rules: [{ kind: "immune", attackTypes: [3] }] },
  { id: "lightning-rod",name: "ひらいしん",     rules: [{ kind: "immune", attackTypes: [3] }] },
  { id: "motor-drive",  name: "でんきエンジン", rules: [{ kind: "immune", attackTypes: [3] }] },
  { id: "sap-sipper",   name: "そうしょく",     rules: [{ kind: "immune", attackTypes: [4] }] },
  { id: "dry-skin",     name: "かんそうはだ",   rules: [{ kind: "immune", attackTypes: [2] },
                                                        { kind: "scale",  attackTypes: [1], factor: 1.25 }] },
  { id: "fluffy",       name: "もふもふ",       rules: [{ kind: "scale",  attackTypes: [1], factor: 2 }] },
  { id: "filter",       name: "フィルター",     rules: [{ kind: "scaleIfSuper", factor: 0.75 }] },
  { id: "solid-rock",   name: "ハードロック",   rules: [{ kind: "scaleIfSuper", factor: 0.75 }] },
  { id: "prism-armor",  name: "プリズムアーマー", rules: [{ kind: "scaleIfSuper", factor: 0.75 }] },
  { id: "wonder-guard", name: "ふしぎなまもり", rules: [{ kind: "wonderGuard" }] },
];
```

- 新しい特性の追加は `ABILITIES` への1行追加で完結する（ロジック変更不要）。
- `attackTypes` の数値は既存 `TYPES` のタイプIDを参照する。

### 5.2 状態の追加

```js
let selectedAbilityId = "none";   // 選択中の特性ID（常に1件）
```

既存の `selectedTypes` と合わせて可変状態は2つとなる。単方向データフロー（状態変更→`renderAll()`）は維持する。

## 6. 関数仕様の変更

### 6.1 新規関数

#### `applyAbility(baseMultipliers, ability)` — 特性補正適用
| 項目 | 内容 |
|---|---|
| 引数 | `baseMultipliers`: number[]（`calcMultipliers` の戻り値、長さ18） / `ability`: ABILITIES の要素 |
| 戻り値 | number[]（補正後の倍率配列、長さ18） |
| 性質 | 純粋関数。入力配列は変更しない |

処理ロジック（4.1の計算順序を実装）:
```
各攻撃タイプ atk について:
  m = baseMultipliers[atk]
  1. rules に immune があり atk が attackTypes に含まれる → 0 を返す
  2. rules の scale のうち atk が対象のもの → m *= factor（複数該当時は順次乗算）
  3. rules に scaleIfSuper があり、かつ m > 1 → m *= factor
  4. rules に wonderGuard があり、かつ baseMultipliers[atk] <= 1 → 0
```

#### `formatMultiplier(value)` — 倍率表記整形
- `4 → "4倍"`、`0.75 → "0.75倍"`、`0.125 → "0.125倍"` のように小数の末尾ゼロを除いた文字列を返す。
- 0 は「無効」と表記する。

### 6.2 既存関数の変更

| 関数 | 変更内容 |
|---|---|
| `calcMultipliers` | **変更なし**（タイプ相性のみを担当。特性補正は `applyAbility` に分離） |
| `groupByMultiplier` | 固定の `MULTIPLIER_GROUPS` 走査をやめ、**結果に実際に含まれる倍率値を降順ソートして動的にグループ化**する方式に変更 |
| `renderResult` | `calcMultipliers(selectedTypes)` → `applyAbility(…, 選択中特性)` → `groupByMultiplier` の順で呼び出す |
| `renderStatus` | 特性セレクトボックスの表示状態を反映 |
| `clearSelection` | タイプに加えて `selectedAbilityId = "none"` にリセットする |
| `renderAll` | 変更なし（呼び出し先の変更のみ） |

### 6.3 定数の変更

- `MULTIPLIER_GROUPS`（固定6グループ定義）を廃止し、以下の2関数に置き換える。
  - `groupLabel(value)`: `formatMultiplier` の結果＋カテゴリ語（下表）を結合した見出しを返す。
  - `groupColor(value)`: 倍率値からアクセント色を返す。

| 倍率範囲 | カテゴリ語 | アクセント色 |
|---|---|---|
| 4以上 | こうかばつぐん | 濃い赤 `#D32F2F` |
| 1より大きく4未満 | こうかばつぐん | 赤 `#F06292` |
| 1 | 等倍 | グレー `#9E9E9E` |
| 0.5以上1未満 | いまひとつ | 青 `#64B5F6` |
| 0より大きく0.5未満 | いまひとつ | 濃い青 `#1976D2` |
| 0 | こうかなし | 黒 `#424242` |

※ 従来の6値（0/0.25/0.5/1/2/4）のみが発生する「特性なし」時は、既存設計と同一の見出し・色・表示順になることを保証する（後方互換）。

## 7. 画面設計の変更

### 7.1 レイアウト変更（選択ステータスセクション）

特性セレクタを選択ステータスセクション内に追加する。

```
├──────────────────────────────────────────┤
│ 選択中のタイプ                  [クリア]     │
│      【 みず 】 × 【 じめん 】               │
│                                          │
│ 特性: [ 特性なし            ▼ ]            │   ← 追加
│ （例: ふゆう選択時）                         │
│ ⓘ ふゆう: じめん技を無効化                   │   ← 適用中の効果説明
├──────────────────────────────────────────┤
```

### 7.2 画面部品の追加

| 部品ID / クラス | 要素 | 内容 |
|---|---|---|
| `#ability-row` | div | 特性選択行のコンテナ（ステータスセクション下部） |
| `#ability-select` | select | 特性ドロップダウン。`ABILITIES` から生成。初期値 "none" |
| `#ability-note` | p | 選択中特性の効果説明文。「特性なし」時は非表示 |

- 特性の効果説明文は `ABILITIES` に `description` プロパティを追加して保持する（例: `"じめん技を無効化"`）。
- セレクトボックスは既存カードデザインに合わせ、角丸・ボーダー・フォーカス時のアクセント色を適用する。
- `change` イベントで `selectedAbilityId` を更新し `renderAll()` を呼ぶ（リアルタイム更新要件 F-03 を踏襲）。

### 7.3 結果表示の変更

- セクションは補正後の倍率値の**降順**に動的生成する（例: 5倍 → 3倍 → 1.5倍 → … → 無効）。
- タイプ未選択時の挙動は従来どおり（案内メッセージのみ。特性のみ選択されていても計算しない）。

## 10. テスト観点（追加分）

| No | 分類 | 観点 | 期待結果 |
|---|---|---|---|
| A1 | 無効化 | ひこう単タイプ＋ふゆう | じめんが0倍のまま（0×無効の重複でも0倍） |
| A2 | 無効化 | いわ単タイプ＋ふゆう | じめんが2倍→0倍（無効化がタイプ相性に優先） |
| A3 | 定率補正 | こおり単タイプ＋あついしぼう | ほのお2倍→1倍、こおり0.5倍→0.25倍 |
| A4 | 定率補正 | くさ＋はがね複合＋あついしぼう | ほのお4倍→2倍 |
| A5 | 複合ルール | みず単タイプ＋かんそうはだ | みず0.5倍→0倍、ほのお1倍→1.25倍（新倍率値の表示） |
| A6 | 条件付き | いわ＋こおり複合＋フィルター | かくとう4倍→3倍、はがね2倍→1.5倍。等倍以下は変化なし |
| A7 | 特殊 | むし＋ゴースト複合＋ふしぎなまもり | ほのお・ひこう・いわ・あく（2倍）以外がすべて無効 |
| A8 | 定率補正 | くさ＋むし複合＋もふもふ | ほのお4倍→8倍（上方向の新倍率値） |
| A9 | UI | 特性選択後にタイプを変更 | 特性選択が維持されたまま再計算される |
| A10 | UI | クリアボタン押下 | タイプと特性の両方が初期状態（特性なし）に戻る |
| A11 | 後方互換 | 「特性なし」で従来ケース（みず＋じめん等） | 既存実装と同一の見出し・色・順序で表示される |
| A12 | 表示 | 新倍率値（3倍・1.5倍・0.125倍等）の見出し | 「3倍（こうかばつぐん）」等、末尾ゼロなしの表記で降順に表示 |

# ポケモンタイプ相性チェッカー 追加設計書（ポケモン特性）

- 作成日: 2026-07-04
- 対象バージョン: ver2（Electron版）
- 前提設計書: [pokemon_type_chart_pokemon_search_design.md](./pokemon_type_chart_pokemon_search_design.md)

## 1. 目的・要件

| No | 要件 |
|----|------|
| F-19 | data/pokemon.json に、各ポケモンが備えている可能性のある特性情報を追加する |
| F-20 | ポケモン選択時（検索候補から選択）、特性セレクトをそのポケモンが持つ可能性のある特性のみに絞り込む |
| F-21 | 可能性のある特性が1つのみの場合、その特性のみ選択状態にし、「特性なし」は表示しない |
| F-22 | タイプ選択に戻った場合・クリア時は、全特性（＋「特性なし」）から選べるようにする |

## 2. データ設計（F-19）

### 2.1 特性の集約方針（重要）

abilities.json が持つのは「攻撃技のタイプのみに依存する防御側特性」16種だけである。
一方、ポケモンの実際の特性の大半（しんりょく・もうか・せいでんき等）はタイプ相性に
影響しない。そこで各ポケモンの特性（通常特性＋隠れ特性）を次のように変換して保存する。

- 相性に影響する特性（abilities.json に定義あり）→ その特性のID（1〜16）
- 相性に影響しない特性 → **ID 0（特性なし）に集約**
- 重複を除去し昇順で保存。結果は必ず1要素以上になる

### 2.2 data/pokemon.json の形式

```json
{
  "_meta": { ... },
  "pokemon": [
    { "no": 1,   "name": "フシギダネ", "types": [4, 7], "abilityIds": [0] },
    { "no": 25,  "name": "ピカチュウ", "types": [3],    "abilityIds": [0, 8] },
    { "no": 479, "name": "ロトム",     "types": [3, 13], "abilityIds": [1] }
  ]
}
```

- フシギダネ: しんりょく／ようりょくそ → どちらも相性に無関係 → `[0]`
- ピカチュウ: せいでんき（無関係）／ひらいしん（隠れ特性） → `[0, 8]`
- ロトム: ふゆうのみ → `[1]`（「特性なし」になる可能性がないため 0 を含まない）

### 2.3 PokeAPI からの取得

生成スクリプト（scripts/generate-pokemon-json.mjs）で、GraphQL の
`pokemonabilities`（is_hidden を問わず全件）を名前・タイプと同時に取得し、
英語特性名 → abilities.json インデックスの対応表で変換する。

## 3. UI設計（F-20 〜 F-22）

特性セレクトの動作を変更する。

| 場面 | 表示内容 | 既定選択 |
|------|---------|---------|
| 初期状態 ／ タイプ手動選択 ／ クリア | 全17種（特性なし + 16の特性） | 特性なし |
| ポケモン検索で選択 | そのポケモンの abilityIds のみ | 先頭（昇順のため 0 があれば特性なし） |

**具体例**:
- フシギダネを選択 → 特性セレクト = 「特性なし」のみ（相性に影響する特性を持たないため）
- ピカチュウを選択 → 「特性なし」「ひらいしん」の2択。既定は「特性なし」
- ロトムを選択 → 「ふゆう」のみ。「特性なし」は表示されない（F-21）
- クリア時・タイプボタン押下時 → 全17種に復帰（F-22）

## 4. 実装設計

| ファイル | 変更内容 |
|----------|---------|
| scripts/generate-pokemon-json.mjs | GraphQLで `pokemonabilities` を取得し、対応表で `abilityIds` に変換 |
| data/pokemon.json | 全1025件に `abilityIds`（1要素以上）を付与して再生成 |
| src/renderer/app.js | `updateAbilityOptions(abilityIds)` で特性セレクトを動的更新（空配列=全特性） |

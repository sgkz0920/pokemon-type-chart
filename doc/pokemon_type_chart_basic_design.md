## 2. システム概要

### 2.1 目的
ポケモンの防御側タイプ（単一または複合）を指定し、攻撃側全18タイプとの相性（ダメージ倍率）を直感的に確認できるツールを提供する。

### 2.2 システム方式
- **単一HTMLファイル方式**を採用する。
  - HTML / CSS / JavaScript を1ファイル（`pokemon_type_chart.html`）に内包する。
  - 外部ライブラリ・CDN・画像ファイル等の外部リソースは一切使用しない。
- 採用理由：
  - 仕様書「3. 技術スタック」の「GoogleDriveに格納したHTML上の操作で完結すること」を満たすため、ネットワークアクセスや外部依存が不要な自己完結構成とする。
  - サーバサイド処理・ビルド工程が不要で、ファイル配布のみで動作する。

### 2.3 動作環境
| 項目 | 内容 |
|---|---|
| 実行環境 | モダンブラウザ（Chrome / Edge / Firefox / Safari 最新版） |
| 配置場所 | Google Drive（ダウンロードまたはプレビューアプリ経由で開く） |
| ネットワーク | 不要（オフライン動作可） |
| サーバ | 不要 |

> **制約事項**: Google Drive はHTMLの直接ホスティング機能を提供していないため、利用時は「ファイルをローカルにダウンロードして開く」または「HTML Viewer系のDrive連携アプリでプレビューする」運用とする。単一ファイル・外部依存なしの構成により、どちらの方法でも動作する。

## 3. アーキテクチャ設計

単一ファイル内を以下の3層に論理分割する。

```
┌─────────────────────────────────────┐
│ 表示層（View）                        │
│  ・タイプ選択パネル描画                 │
│  ・選択ステータス描画                   │
│  ・結果パネル描画（倍率別グルーピング）    │
├─────────────────────────────────────┤
│ ロジック層（Logic）                    │
│  ・選択状態管理（最大2タイプ、トグル、FIFO）│
│  ・相性倍率計算（タイプ1×タイプ2）        │
│  ・特性補正（applyAbility）             │
│  ・倍率別グルーピング（動的）             │
├─────────────────────────────────────┤
│ データ層（Data）                       │
│  ・タイプ定義（18種：ID、和名、イメージカラー）│
│  ・タイプ相性表（18×18、第6世代以降準拠）   │
│  ・特性定義（宣言的ルール）               │
└─────────────────────────────────────┘
```

- データ層は定数（イミュータブル）として定義し、ロジック層・表示層から参照のみ行う。
- 状態は「選択中タイプの配列（最大2件）」と「選択中特性ID（1件）」のみとし、状態変更のたびに表示層を全再描画する単方向データフローとする。

## 4. 機能一覧

| 機能ID | 機能名 | 概要 | 仕様書対応 |
|---|---|---|---|
| F-01 | タイプ選択機能 | 18タイプボタンから最大2つ選択。再クリックで解除、3つ目選択時は最古の選択を解除（FIFO） | 2.1 |
| F-02 | 相性計算機能 | 攻撃18タイプ×防御タイプ（1〜2）の倍率を算出（0 / 0.25 / 0.5 / 1 / 2 / 4倍） | 2.2 |
| F-03 | 結果表示機能 | 倍率ごとにセクション分けして表示。タイプ名はイメージカラーで色付け。選択操作のたびに即時再描画 | 2.3 |
| F-04 | 選択ステータス表示 | 現在の選択タイプ（1つ目・2つ目）を大きく表示 | 2.4 |
| F-05 | クリア機能 | 選択状態をすべてリセット | 2.4 |

## 5. 画面設計（概要）

画面は1画面のみ。上から「タイプ選択パネル」「選択ステータス」「結果表示パネル」を縦に配置する。

```
┌──────────────────────────────────────────┐
│  ポケモンタイプ相性チェッカー（タイトル）        │
├──────────────────────────────────────────┤
│ タイプ選択パネル                             │
│ [ノーマル][ほのお][みず][でんき][くさ][こおり]  │
│ [かくとう][どく][じめん][ひこう][エスパー][むし] │
│ [いわ][ゴースト][ドラゴン][あく][はがね][フェアリー]│
├──────────────────────────────────────────┤
│ 選択ステータス            [クリア]            │
│      【 みず 】 × 【 じめん 】               │
├──────────────────────────────────────────┤
│ 結果表示パネル                              │
│ ▼ 4倍（こうかばつぐん）                      │
│   [くさ]                                  │
│ ▼ 2倍（こうかばつぐん）                      │
│   [こおり] ...                             │
│ ▼ 1倍（等倍）                              │
│   [ノーマル] ...                           │
│ ▼ 0.5倍（いまひとつ） / 0.25倍 / 無効（0倍）   │
│   ...                                    │
└──────────────────────────────────────────┘
```

- **UIデザイン方針（モダンUI要件）**:
  - CSSカスタムプロパティ・Flexbox/Grid を用いたカード型レイアウト。
  - タイプボタンは角丸・タイプカラー背景・ホバー/選択時のアニメーション（スケール・シャドウ）を適用。
  - スマートフォン幅にも対応するレスポンシブ設計とする。

## 6. データ設計（概要）

| データ名 | 形式 | 内容 |
|---|---|---|
| タイプ定義 | 定数配列（18要素） | タイプID、和名、イメージカラー（16進カラーコード） |
| タイプ相性表 | 定数2次元配列（18×18） | 攻撃タイプ×防御タイプの倍率（0 / 0.5 / 1 / 2）。第6世代以降の相性に準拠 |
| 選択状態 | 可変配列（0〜2要素） | 選択中の防御タイプIDを選択順に保持 |

詳細（相性表の全値・カラーコード等）は詳細設計書に記載する。

# ポケモンタイプ相性チェッカー 追加設計書（ポケモン名検索）

- 作成日: 2026-07-04
- 対象バージョン: ver2（Electron版）
- 前提設計書: [pokemon_type_chart_electron_design.md](./pokemon_type_chart_electron_design.md)

## 1. 目的・要件

| No | 要件 |
|----|------|
| F-16 | ポケモン名での検索を可能にする（該当ポケモンのタイプが防御側タイプとして設定される） |
| F-17 | タイプ選択とポケモン名指定をタブで切り替えられるようにする。「選択中のタイプ」「相性倍率」エリアは従来のまま共用する |
| F-18 | ポケモンデータはJSON形式の独立ファイルで保存する |

## 2. データ設計（F-18）

### 2.1 data/pokemon.json

```json
{
  "_meta": { "createdAt": "...", "name": "...", "description": "...", "fileName": "pokemon.json" },
  "pokemon": [
    { "no": 1, "name": "フシギダネ", "types": [4, 7] }
  ]
}
```

- `no`: 全国図鑑番号（昇順・一意）
- `name`: 日本語名（カタカナ）
- `types`: タイプID配列（types.jsonのIDに対応。1〜2件）

### 2.2 データの出所と生成

- [PokeAPI](https://pokeapi.co/) のGraphQLエンドポイントから、全種の日本語名
  （ja-Hrkt）と基本フォルムのタイプを一括取得し、`scripts/generate-pokemon-json.mjs`
  で `data/pokemon.json` に変換する。生成スクリプトはリポジトリに含め、
  新世代追加時に再実行して更新できるようにする。
- 対象は各種の**基本フォルム（is_default）のみ**。リージョンフォーム
  （アローラ・ガラル等）やメガシンカ・フォルム違いは対象外（将来拡張）。
- タイプ名（英語）→タイプIDの対応表は生成スクリプト内に持つ。

## 3. UI設計（F-17）

「防御側タイプを選択」セクションの先頭にタブを設け、2つの選択方式を切り替える。

```
┌─────────────────────────────────┐
│ [タイプで選択] [ポケモン名で検索]   ← タブ       │
│ （タブに応じて下記のどちらかを表示）              │
│  A: タイプボタン18個（従来どおり）               │
│  B: 検索入力欄 ＋ 候補リスト                    │
├─────────────────────────────────┤
│ 選択中のタイプ（従来のまま・共用）               │
├─────────────────────────────────┤
│ 相性倍率 ＋ クリア（従来のまま・共用）           │
└─────────────────────────────────┘
```

- タブを切り替えても選択状態（選択中のタイプ・特性・相性倍率）は変化しない。
- 候補リストの項目: 図鑑No・ポケモン名・タイプバッジ（1〜2個）。
- 候補をクリックすると、そのポケモンのタイプが防御側タイプに設定され
  （既存選択は置き換え）、「選択中のタイプ」エリアへ自動スクロールする
  （タイプ選択のS1と同じ導線）。選択された候補はハイライト表示する。
- タイプボタンでの選択変更・クリアを行うと候補ハイライトは解除する。
  クリア時は検索入力・候補リストもリセットする。

## 4. 検索仕様（F-16）

- 入力のたびに逐次検索（インクリメンタルサーチ）。
- ひらがな入力はカタカナに正規化して照合する（例: 「ふしぎだね」→「フシギダネ」）。
  前後の空白は無視。英数字はそのまま照合。
- 一致方式: 部分一致。**前方一致を優先**して図鑑番号順に表示する。
- 表示件数は最大20件（超過時は「絞り込んでください」の注記を表示）。
- 0件時は「見つかりません」を表示。空文字時は候補を表示しない。

## 5. モジュール設計

| ファイル | 変更内容 |
|----------|----------|
| data/pokemon.json | 新規。全国図鑑の基本フォルム全種 |
| scripts/generate-pokemon-json.mjs | 新規。PokeAPI GraphQLからpokemon.jsonを生成 |
| src/main/main.js | pokemon.jsonも読み込んでレンダラーへ渡す |
| src/renderer/logic.js | `normalizeKana(text)`・`searchPokemon(pokemon, query, limit)` を追加（純粋関数） |
| src/renderer/index.html | タブUI・検索入力欄・候補リストの構造を追加 |
| src/renderer/style.css | タブ・検索欄・候補リストのスタイル（ライト/ダーク両対応・スマホ対応） |
| src/renderer/app.js | タブ切替、検索イベント、候補選択→タイプ設定、fetchフォールバックにpokemon.json追加 |

## 6. テスト計画

### 6.1 単体テスト（追加）

| No | 検証内容 |
|----|----------|
| U7 | pokemon.jsonの整合性: 1000件以上、図鑑Noが一意・昇順、名前が非空、typesが1〜2件かつ0〜17、代表値（フシギダネ=くさ×どく、リザードン=ほのお×ひこう等） |
| U8 | normalizeKana: ひらがな→カタカナ変換、空白除去、カタカナ・英数字はそのまま |
| U9 | searchPokemon: ひらがな入力での一致、前方一致優先、部分一致、0件、空文字、上限件数 |

# ポケモンタイプ相性チェッカー 追加設計書（モバイルUI改善・不具合修正）

- 作成日: 2026-07-04
- 対象バージョン: ver2（Electron版）
- 前提設計書: [pokemon_type_chart_electron_design.md](./pokemon_type_chart_electron_design.md)

## 1. 目的

スマートフォン操作時に確認された下記5件の事象・要望に対応する。

| No | 区分 | 内容 |
|----|------|------|
| F-10 | 仕様変更 | テーマ切替ボタンをスマホでは画面追随（fixed）ではなくページ最上部に固定する |
| F-11 | 不具合修正 | クリア後に再度タイプを選択した際、「選択中のタイプ」エリアへスクロールしないことがある |
| F-12 | 仕様追加 | ダブルタップ等によるブラウザのズーム（拡大縮小）を抑止する |
| F-13 | 仕様追加 | 見出し「ポケモンタイプ相性チェッカー」を常に1行で表示する |
| F-14 | 仕様追加 | 説明文を常に2行で表示する |

## 2. 各項目の設計

### 2.1 F-10: テーマ切替ボタンの配置変更

- 現状: `#theme-toggle` は `position: fixed`（全画面幅共通）で、スクロールしても画面右上に追随する。
- 変更: 幅600px以下（スマホ向けメディアクエリ内）では `position: absolute` とし、
  ページ（文書）最上部の右上に配置する。スクロールするとボタンはページと一緒に画面外へ流れる。
- デスクトップ（幅601px以上）は従来どおり `fixed` を維持する（要望はスマホ操作時のみのため）。

### 2.2 F-11: 自動スクロール不発の修正

- 事象: タイプ選択→「選択中のタイプ」へスクロール→クリア（上部へスクロール）→
  直後に再度タイプを選択すると、「選択中のタイプ」エリアへスクロールしないことがある。
- 原因: `scrollToPanelIfNeeded()` は「対象パネルが画面内に完全に収まっているか」を
  呼び出し時点の座標で判定している。クリアによる上方向スムーズスクロールの
  アニメーション進行中に再選択すると、その時点ではまだ「選択中のタイプ」エリアが
  画面内にあるためスクロール不要と誤判定され、その後アニメーションが上端まで
  進んでエリアが画面外に出る。
- 対策: スムーズスクロールの「目標パネル」をモジュール変数 `pendingScrollPanel` に記録し、
  **別パネルへのスクロール進行中**の要求（割り込み）を特別扱いする。
  - 割り込み時に対象パネルが画面外（一部でも）の場合: `scrollIntoView` する
    （移動量が0でないため進行中のアニメーションは正しく置き換えられる）。
  - 割り込み時に対象パネルが画面内に完全に収まっている場合: 進行中のアニメーションを
    中断してその位置に留める。同位置への `scrollTo` / `scrollIntoView` は
    Chromiumでは無視され進行中のアニメーションを中断できないため、
    instantで1pxずらして元の位置に戻すことで確実に中断する。
  - スクロール完了は `scroll` イベントのデバウンス（150ms間イベントなし）で検知し、
    `pendingScrollPanel` を解除する。（`scrollend` イベントはiOS Safari非対応のため使わない）
  - 進行中でない場合の挙動は従来どおり（画面内に完全に収まっていればスクロールしない＝PCでは実質無効）。

### 2.3 F-12: 画面ズームの抑止

3層で抑止する（環境により有効な手段が異なるため併用する）。

| 手段 | 対象 | 内容 |
|------|------|------|
| viewport meta | 全般 | `maximum-scale=1.0, user-scalable=no` を追加 |
| CSS | ダブルタップズーム | `body { touch-action: manipulation; }`（タップ遅延も解消） |
| JS | iOS Safariのピンチズーム | `gesturestart` イベントを `preventDefault()`（iOSはviewport指定を無視するため） |

### 2.4 F-13: 見出しの1行固定

- `header h1` に `white-space: nowrap` を指定する。
- スマホ幅では文字サイズを画面幅連動で縮小し、はみ出しを防ぐ:
  `font-size: clamp(0.7rem, calc((100vw - 136px) / 15), 1.25rem)`
  - `136px` = body左右padding 16px + header左右padding 104px（テーマボタン回避用52px×2）
    + スクロールバー余裕分16px（100vwはスクロールバー幅を含むため）
  - `15` = 見出し14文字 + letter-spacing（実測14.7em相当）+ 余裕分

### 2.5 F-14: 説明文の2行固定

- 説明文を意味の切れ目で2つの `span.desc-line` に分割し、各spanを
  `display: block; white-space: nowrap` とする（全環境で常に2行）。
  - 1行目: 「防御側のタイプを選ぶと、攻撃側全18タイプとの」
  - 2行目: 「相性倍率を表示します（第6世代以降準拠）」
- スマホ幅では画面幅連動で縮小: `font-size: clamp(0.5rem, calc((100vw - 136px) / 22.5), 0.85rem)`
  - `22.5` = 最長行（1行目）の実効幅（実測22.1em相当）+ 余裕分

## 3. 変更対象ファイル

| ファイル | 変更内容 |
|----------|----------|
| src/renderer/index.html | viewport meta変更、説明文のspan分割 |
| src/renderer/style.css | F-10（メディアクエリ内absolute）、F-12（touch-action）、F-13/F-14（nowrap・clamp） |
| src/renderer/app.js | F-11（pendingScrollPanel導入）、F-12（gesturestart抑止） |
| scripts/e2e.mjs | E6〜E9の検証追加 |

データファイル（data/*.json）、ロジック層（logic.js）、メインプロセスに変更はない。

## 4. テスト計画

単体テスト（U1〜U6）は変更対象外のため既存24件の回帰確認のみ。E2Eに以下を追加する。

| No | 検証内容 |
|----|----------|
| E6 | モバイル幅（390px）で `#theme-toggle` が `position: absolute` になり、スクロール後に画面外へ流れる。デスクトップ幅では `fixed` のまま |
| E7 | モバイル幅で見出しが1行表示（折り返し・はみ出しなし） |
| E8 | 説明文が常に2行（`.desc-line` が2要素、モバイル幅ではみ出しなし） |
| E9 | クリア直後（スクロールアニメーション中）に再度2タイプを選択しても「選択中のタイプ」エリアへスクロールする（F-11回帰） |

ズーム抑止（F-12）は実機タッチ操作が必要なため、viewport metaの内容検証のみE2Eで行う。

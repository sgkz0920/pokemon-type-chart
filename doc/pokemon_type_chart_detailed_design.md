## 2. ファイル構成

成果物は単一ファイル `pokemon_type_chart.html` とし、内部構造は以下とする。

```
pokemon_type_chart.html
├─ <head>
│   ├─ meta（charset=UTF-8, viewport）
│   ├─ <title>ポケモンタイプ相性チェッカー</title>
│   └─ <style>          … 全CSS（インライン内包）
└─ <body>
    ├─ <header>          … タイトル
    ├─ <section id="type-panel">    … タイプ選択パネル
    ├─ <section id="status-panel">  … 選択ステータス＋クリアボタン
    ├─ <section id="result-panel">  … 結果表示パネル
    └─ <script>          … 全JavaScript（インライン内包）
        ├─ データ定義（TYPES / TYPE_CHART）
        ├─ 状態管理（selectedTypes）
        ├─ ロジック関数（toggleType / clearSelection / calcMultipliers / groupByMultiplier）
        └─ 描画関数（renderTypePanel / renderStatus / renderResult / renderAll）
```

外部リソース（CDN・画像・Webフォント）への参照は禁止とする。

## 4. 関数仕様

### 4.1 ロジック関数

#### `toggleType(typeId)` — タイプ選択トグル（F-01）
| 項目 | 内容 |
|---|---|
| 引数 | `typeId`: number（0〜17） |
| 戻り値 | なし（`selectedTypes` を更新し `renderAll()` を呼ぶ） |

処理ロジック:
1. `typeId` が `selectedTypes` に**含まれる**場合 → 配列から削除する（選択解除）。
2. 含まれず、`selectedTypes.length < 2` の場合 → 末尾に追加する。
3. 含まれず、`selectedTypes.length === 2` の場合 → **先頭（最古）を削除**し、末尾に `typeId` を追加する（FIFO）。
4. `renderAll()` を呼び出す。

#### `clearSelection()` — 選択クリア（F-05）
1. `selectedTypes = []` にリセットする。
2. `renderAll()` を呼び出す。

#### `calcMultipliers(defenseTypeIds)` — 相性倍率計算（F-02）
| 項目 | 内容 |
|---|---|
| 引数 | `defenseTypeIds`: number[]（長さ1〜2の防御タイプID配列） |
| 戻り値 | number[]（長さ18。インデックス=攻撃タイプID、値=最終倍率） |
| 性質 | 純粋関数（状態を参照・変更しない）。将来の特性補正・逆引き拡張の共通基盤とする |

処理ロジック:
```
各攻撃タイプ atk (0..17) について:
    multiplier = defenseTypeIds.reduce(
        (acc, def) => acc * TYPE_CHART[atk][def], 1)
```
- 単タイプ時は相性表の値がそのまま、複合タイプ時は2値の積となり、結果は必ず `{0, 0.25, 0.5, 1, 2, 4}` のいずれかに収束する。

#### `groupByMultiplier(multipliers)` — 倍率別グルーピング（F-03）
| 項目 | 内容 |
|---|---|
| 引数 | `multipliers`: number[]（`calcMultipliers` の戻り値） |
| 戻り値 | `Map<number, number[]>`（キー=倍率、値=攻撃タイプID配列。キーは 4→2→1→0.5→0.25→0 の順） |

- 該当タイプが0件の倍率グループは戻り値に含めない（表示しない）。

> **改訂（特性対応）**: 特性補正により任意の倍率値（3倍・1.5倍・0.125倍等）が発生するため、キーは「実際に発生した倍率値の降順」に変更。あわせて `applyAbility`・`formatMultiplier`・`groupLabel`・`groupColor` を追加。詳細は追加設計書（pokemon_type_chart_ability_design.md）4〜6章を参照。

### 4.2 描画関数

| 関数 | 責務 |
|---|---|
| `renderTypePanel()` | 初期化時に18個のタイプボタンを生成し `#type-panel` に配置。クリックイベントに `toggleType` を紐付け。再描画時は選択中ボタンに `.selected` クラスを付与/除去 |
| `renderStatus()` | `selectedTypes` を「タイプ1 × タイプ2」形式で `#status-panel` に大きく表示。未選択スロットはプレースホルダ（例:「タイプを選択」）を表示 |
| `renderResult()` | `selectedTypes` が空なら案内メッセージを表示。1件以上なら `calcMultipliers` → `groupByMultiplier` の結果を倍率セクション単位で描画 |
| `renderAll()` | 上記3関数を順に呼び出す（状態変更後の一括再描画） |

### 4.3 初期化処理

`DOMContentLoaded` 時に以下を実行する。
1. `renderTypePanel()` によるボタン生成。
2. クリアボタンに `clearSelection` を紐付け。
3. `renderAll()` による初期描画（未選択状態）。

### 5.2 倍率セクションの表示仕様

表示順は効果の大きい順で固定とする。

| 倍率 | 見出しラベル | アクセント色（見出し・枠） |
|---|---|---|
| 4 | 4倍（こうかばつぐん） | 濃い赤 `#D32F2F` |
| 2 | 2倍（こうかばつぐん） | 赤 `#F06292` |
| 1 | 1倍（等倍） | グレー `#9E9E9E` |
| 0.5 | 0.5倍（いまひとつ） | 青 `#64B5F6` |
| 0.25 | 0.25倍（いまひとつ） | 濃い青 `#1976D2` |
| 0 | 無効（こうかなし） | 黒 `#424242` |

- 未選択時: セクションを表示せず「防御側のタイプを選択してください」の案内メッセージのみ表示する。

> **改訂（特性対応）**: 上表の固定6セクション方式は廃止し、倍率値の範囲からラベル・色を導出する動的方式に置換（「特性なし」時の表示は上表と同一になることを保証）。規則は追加設計書 6.3 を参照。

### 5.3 スタイル方針（モダンUI要件）

- CSSカスタムプロパティでタイプカラー・余白・角丸を一元管理する。
- カード型レイアウト（各セクションを白背景＋シャドウのカードとする。ページ背景は淡いグラデーション）。
- タイプボタンは `transition` によるホバー時の浮き上がり（`transform: translateY` + `box-shadow`）を適用する。
- 明度の高いタイプカラー（でんき・こおり・はがね・フェアリー等）は文字色を暗色にし、コントラストを確保する。
- `@media (max-width: 600px)` でグリッド列数・フォントサイズを縮小するレスポンシブ対応を行う。

## 6. 処理フロー

```
[ページ読込]
    → 初期化（ボタン生成・イベント登録）→ renderAll()（未選択表示）

[タイプボタンクリック]
    → toggleType(typeId)
        ├ 選択済み       → 解除
        ├ 選択数 < 2     → 追加
        └ 選択数 = 2     → 最古を解除して追加（FIFO）
    → renderAll()
        ├ renderTypePanel()  … ボタン選択状態の反映
        ├ renderStatus()     … 選択バッジの更新
        └ renderResult()     … calcMultipliers → groupByMultiplier → 描画

[クリアボタンクリック]
    → clearSelection() → renderAll()（未選択表示に戻る）
```

すべての操作が同期処理であり、クリックごとに即時再描画される（F-03 リアルタイム更新要件を満たす）。

## 7. テスト観点

| No | 分類 | 観点 | 期待結果 |
|---|---|---|---|
| 1 | 選択 | 未選択状態でタイプAをクリック | Aが1つ目として選択される |
| 2 | 選択 | 選択済みのAを再クリック | Aが解除され未選択に戻る |
| 3 | 選択 | A選択中にBをクリック | A・Bの2タイプ選択となる |
| 4 | 選択 | A→B選択中にCをクリック | Aが解除され、B・Cの選択となる（FIFO） |
| 5 | 選択 | A→B選択中にAを再クリック | Aのみ解除され、Bが残る |
| 6 | 計算 | 単タイプ「はがね」を選択 | あく・ゴーストが1倍（第6世代準拠）、かくとう・じめん・ほのおが2倍、どくが0倍 |
| 7 | 計算 | 複合「みず＋じめん」を選択 | くさが4倍、でんきが0倍、ほのお・どく・いわ・はがねが0.5倍 |
| 8 | 計算 | 複合「ゴースト＋ノーマル」を選択 | ノーマル・かくとう・ゴーストが0倍（ゴースト攻撃はノーマルに無効のため2×0=0）、あくのみ2倍 |
| 9 | 表示 | 各倍率グループの表示 | 4→2→1→0.5→0.25→0 の順で、該当0件のグループは非表示 |
| 10 | 表示 | タイプ名の色 | 全タイプが規定のイメージカラーで表示される |
| 11 | クリア | 2タイプ選択後クリア | 未選択状態＋案内メッセージ表示に戻る。未選択時クリアボタンは非活性 |
| 12 | 環境 | オフラインでファイルを開く | 外部通信なしで全機能が動作する |
